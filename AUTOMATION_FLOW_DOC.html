Automation Flow

This document describes the compare_file_refs automation at a high level. It focuses on the business flow, system handoffs, and exception paths rather than code-level implementation.

Figure 1: High-level reconciliation and exception-management flow.

Purpose

The automation reconciles LegalSuite matters against Standard Bank source files, identifies exceptions, investigates weekly-balancing anomalies, prepares correction files, and optionally closes matters in LegalSuite when the weekly status supports closure.

Main Systems

System Role
LegalSuite Source of matter records, including Matter File Ref, Their Ref, and current matter state.
FTP Source of claim and weekly files, and destination for the matter-ref correction workbook.
Claim Amount File Daily file used for the primary comparison against LegalSuite matter refs.
Weekly Balancing File Operational status source used to resolve missing refs and identify closure actions.
Verification Outputs Audit artifacts showing what the automation concluded and what files were produced.

End-to-End Flow

  1. Collect the matter list from LegalSuite or from a CSV input.
  2. Download the current Standard Bank claim amount file from FTP.
  3. Compare LegalSuite Matter File Ref values with claim file File Ref values.
  4. Write the missing refs to an exception workbook.
  5. If exceptions exist, download the current weekly balancing file.
  6. Search weekly balancing by File Ref.
  7. If still not found, search weekly balancing by LegalSuite Their Ref against the weekly Matter column.
  8. Create downstream outputs based on what is found and what status is returned.
  9. Upload the matter-ref correction file to FTP when a weekly mismatch is confirmed.
  10. Optionally update LegalSuite for matters whose weekly status is Closed.
  11. Create verification workbooks and run summaries.

Decision Paths

Direct Match In Claim File

If the LegalSuite matter ref exists in the claim file, the record is considered reconciled and no exception action is needed.

Missing From Claim File, Found In Weekly By File Ref

If the matter is missing from the claim file but appears in weekly balancing under the same File Ref, the weekly status is used for downstream handling.

Missing From Claim File, Resolved By Their Ref

If the matter cannot be found in weekly balancing by File Ref, the automation fetches the LegalSuite matter, reads Their Ref, and searches the weekly Matter column using that value.

A successful match means the weekly record exists but is carrying the wrong file reference. The automation then creates Panel L Matter Ref Updates.xlsx with:

  • Matter File Ref: the correct LegalSuite file ref
  • Their Ref: the account number used to locate the weekly row
That workbook is uploaded to the FTP folder Matter Ref Updates itself, not to any subfolder.

Missing From Claim File, Not Found In Weekly At All

If the matter cannot be found in weekly balancing by either File Ref or Their Ref, it remains unresolved for the run and is written to the not-found output for manual follow-up.

Status Handling

Weekly balancing is treated as the operational status source during exception handling.

  • If the weekly status is open or live-type, the matter generally remains live in LegalSuite.
  • If the weekly status is closed, the automation can optionally trigger the LegalSuite closure flow.

LegalSuite Closure Logic

  1. Identify missing matters whose weekly status is Closed.
  2. Fetch the matter from LegalSuite by File Ref.
  3. Attempt the archive update.
  4. Fetch the matter again and verify the returned archive state.
  5. If archive is rejected or does not stick, fall back to a Pending Deletion state.
  6. Fetch the matter again and verify the fallback state.

Operational Outputs

  • Report (SUP).xlsx
  • Missing_Matter_File_Ref.xlsx
  • Matter_Found_From_Weekly.xlsx
  • Matter_Not_Found_From_Weekly.xlsx
  • Panel L Matter Ref Updates.xlsx

Verification Outputs

  • verification/ copies of generated workbooks
  • verification_summary.json
  • verification_summary.txt

Control Principles

  • Prefer direct reconciliation first.
  • Use weekly balancing as the primary exception-resolution source.
  • Use Their Ref as the fallback key when File Ref fails.
  • Do not close matters simply because they are absent from the claim file.
  • Keep an audit trail for every significant result.

Summary

At an abstract level, this automation is an exception-management pipeline: compare by file ref, validate against weekly balancing, resolve anomalies through account number fallback, generate correction files where identifiers are inconsistent, and only update LegalSuite when the weekly status supports that action.
